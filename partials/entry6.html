Unit Testing - Backend

For the back-end, our goal was to create unit tests for all our different components:

  • models
  • views
  • urls
  • serializers

We were successfully able to write tests that makes sure that our models (food, meal_food, location, location_menu) were properly being serialized.

For that, we created:

  • food and locations objects (that we populated with dummy data).
  • using the food objects, I created a meal_food object.
  • using the location objects and meal_food object , we created a Location_menu object

We made sure that:

  • each serializer returned a json object (i.e a list of key value pairs)
  • the keys returned were consistent with the models’ fields
  • for the meal_food serializer, the length of the values corresponding to key “foods” was consistent with the amount of Food objects I initially created

We unfortunately had a harder time testing the url/ views component.

Our goal was to do a reverse match of urls by calling “reverse match” on the name of the base_name and the corresponding arguments, and expecting to get the appropriate url.

However, we kept getting errors as the interpreter didn’t recognize the pairing (and gave us a no match)

From the reading we have done online, it looks the error is coming from the fact that our urls are not in the urlpatterns but are registered to the default router.

Side note: these unit tests do not correspond to the updated version of our app but the version prior to spring break.
