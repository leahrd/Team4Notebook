Entry 8: Optimization

March 30, 2017

Optimization

Before attempting to complete any static content optimization, we used the YSlow! browser extension to quantify the performance of our app. We received an initial grade, C. From the start we had avoided some issues that tend to slow a web app way down but we definitely had some improvements to make. Most of our issues were the same as those covered in class. We really needed to combine our JS and CSS files as well as minify them. Before any optimization our JavaScript files were not at the bottom, an easy fix. We didn’t have any expiry headers, and although we don’t have any images, adding these would help improve our app’s performance on any subsequent visits a specific user might make. Additionally we could improve our performance by using CDN’s.

At first we attempted to minify our JS and CSS files using minifier.org without realizing that Angular has some really complicated dependencies. So, in order to properly minify our files, we needed to pre-configure dependencies before the files could be minified using ng-annotate. We used Grunt to minify and concatenate our JavaScript and CSS files. We also put our Javascript files at the end of the body. In the end, we were able to improve our YSlow grade to an A.

We are still looking into how to use CDN and setting expiry headers. Once we figure these out we think our app will me much faster, but we have found our app is already noticeably faster.
